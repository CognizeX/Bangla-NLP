### Project Initiative Overview

This document presents **20 curated NLP projects** designed for undergraduate students at AIUB, focusing on **Bengali language processing** and **low-resource language challenges**. The projects span four distinct tracks:

- **Bengali NLP Core** (7 projects): Sentiment analysis, machine translation, speech processing
- **Startup Track** (6 projects): Revenue-generating products with real-world applications
- **Research Track** (4 projects): Publishable research, benchmarking, endangered languages
- **Advanced/Multimodal** (3 projects): Cutting-edge AI, vision-language systems, LLM training

### Key Objectives

[CHECK] **Deep Technical Understanding**: Students must explain every algorithm (to a 10-year-old AND a professor)  
[CHECK] **Real-World Impact**: Build prototypes with live demos, not just research papers  
[CHECK] **Measurable Innovation**: Compare against 2+ baselines; implement meaningful modifications  
[CHECK] **Founder Mentality**: Think about users, revenue models, and scaling to 100K users  
[CHECK] **Production Quality**: Reproducible code, full documentation, clean architecture  

### Why These Projects Matter

- **230M+ Bengali speakers** with near-zero NLP tools
- Projects address **real social problems**: legal access, education, mental health, agriculture
- **Startup potential**: Multiple projects have clear $50K-$300K MRR revenue models
- **Research value**: Several projects are publishable at ACL/EMNLP
- **Career launch**: Success here leads to industry positions (precedent: BanglaBERT author → Amazon AI)

## **1. BanglaSentiment Pro — Multi-domain Sentiment & Emotion Analyser**

[BanglaBERT](https://huggingface.co/csebuetnlp/banglabert) | [Streamlit Demo](https://streamlit.io) | [FastAPI Guide](https://fastapi.tiangolo.com)

**Category**: `BENGALI` + `STARTUP` | **Difficulty**: ⭐⭐ (Medium)

**Problem Statement**:

- Sentiment analysis in Bengali is severely understudied
- E-commerce platforms (Daraz, Shajgoj, Chaldal) have 50M+ Bengali reviews
- No production-grade Bengali sentiment tool exists
- Existing models fail on Banglish, sarcasm, and dialectal variations

**What to Build**:

- [ ] Scrape 50K+ reviews from Daraz, Shajgoj, Chaldal
- [ ] Fine-tune BanglaBERT for 6 emotion classes (joy, anger, fear, sadness, surprise, disgust)
- [ ] Add sarcasm detection module
- [ ] Implement Banglish support via transliteration
- [ ] Build browser extension to overlay sentiment on Bengali text

**System Requirements**:

- **GPU**: NVIDIA GPU with 8GB+ VRAM (T4 minimum recommended)
- **RAM**: 16GB
- **Storage**: 50GB for dataset + model weights
- **Python**: 3.8+
- **Estimated Training Time**: 4-6 hours

**Dataset Type**:

- Multi-label classification (6 emotions + sentiment scale)
- Bengali e-commerce reviews with manual annotations
- ~50,000 samples minimum

**Data Sources**:

- 📍 Daraz.com.bd (Bengali reviews - web scraping)
- 📍 Shajgoj.com (Fashion reviews)
- 📍 Chaldal.com (Grocery reviews)
- 📍 HuggingFace: BanglaBERT pre-trained dataset
- 📍 Manual annotation campaigns (mechanical turk or student volunteers)

**Tech Stack**:

- BanglaBERT (for transformer backbone)
- HuggingFace Transformers (fine-tuning)
- Streamlit (demo dashboard)
- FastAPI (REST backend)
- Chrome Extension API (browser plugin)

**Modifications to Baseline**:
[DONE] Multi-task learning (emotion + sentiment simultaneously)
[DONE] Contrastive loss for sarcasm detection
[DONE] Custom Banglish normaliser using phonetic mapping
[DONE] Domain adaptation for 3+ e-commerce platforms

**Revenue Model**:

- **B2B API**: $0.001/call for e-commerce platforms
- **Expected MRR**: 10M calls/month = $10K starting
- **Pricing Tiers**: Pay-as-you-go, monthly subscriptions

**Success Criteria**:

- F1 ≥ 0.85 on held-out test set (vs. 0.72 for mBERT baseline)
- Sarcasm detection accuracy ≥ 80%
- Live demo accepting real Bengali text
- GitHub repo with full documentation

**Risks & Mitigation**:

- ⚠️ **Risk**: Data imbalance (some emotions rare)  
  → **Solution**: Use focal loss, weighted sampling
- ⚠️ **Risk**: Banglish handling complexity  
  → **Solution**: Pre-build phonetic mapping rules, augment data

**Estimated Effort**: 6-8 weeks | **Team Size**: 2-3 people | **Skills**: ML, web scraping, API design
